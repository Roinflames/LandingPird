    Usuarios

  • GET /api/user
      • Petición HTTP que retorna la totalidad de usuarios registrados en el servidor.
  • GET /api/user/:user_id
      • Petición HTTP que retorna el usuario correspondiente al ID único indicado en la URI.
  • POST /api/user
      • Petición HTTP que requiere los parámetros name (email) y password en el header de esta. Con estas credenciales se crea un registro en la base de datos con ID único, con el cual el usuario podrá loguear y utilizar la aplicación.
  • PUT /api/user/:user_id
      • Petición HTTP que actualiza la información de un usuario registrado mediante los parámetros indicados en los headers de la misma, para modificar a un usuario específico se debe indicar su ID único en la URI.
  • DELETE /api/user/:user_id
      • Petición HTTP que borra la información de un usuario registrado con ID único indicado en la URI.

    Dispositivos

  • GET /api/device
      • Petición HTTP que retorna la totalidad de dispositivos registrados en el servidor.
  • GET /api/device/:device_id
      • Petición HTTP que retorna el dispositivo correspondiente al ID único indicado en la URI.
  • POST /api/device
      • Petición HTTP que requiere los parámetros name (email) y password en el header de esta. Con estas credenciales se crea un registro en la base de datos con ID único, con el cual el dispositivo podrá autenticarse y registrar mediciones.
  • PUT /api/device/:device_id
      • Petición HTTP que actualiza la información de un dispositivo registrado mediante los parámetros indicados en los headers de la misma, para modificar a un dispositivo específico se debe indicar su ID único en la URI.
  • DELETE /api/device/:device_id
      • Petición HTTP que borra la información de un dispositivo registrado con ID único indicado en la URI.

    Autenticación

  • POST /api/signin
      • Petición HTTP que por medio del email y password presentes en el header de la misma autentica un usuario para permitir el acceso a la aplicación, el valor de retorno es un token de autenticación.
  • POST /api/signup
      • Petición HTTP que por medio del email y password presentes en el header de la misma crea un registro de usuario en la base de datos con ID único para permitir la autenticación de este usuario.
  • POST /api/google
      • Petición HTTP.
  • POST /api/googleNative
      • Petición HTTP.
  • POST /api/deviceSignin
      • Petición HTTP.
